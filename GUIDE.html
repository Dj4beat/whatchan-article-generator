2. How it works (architecture)

Your browser never talks to OpenRouter directly. Everything goes through a Cloudflare Worker that holds your API key securely.

Your Browser (GitHub Pages)
    │
    ├── GET  /api/rss           → Cloudflare Worker → RSS feeds (BBC, Sky, etc.)
    ├── POST /api/fetch-article → Cloudflare Worker → Article URL (extracts text)
    └── POST /api/messages      → Cloudflare Worker → OpenRouter API → AI model

This means:

  • Your OpenRouter API key is never visible in the browser or page source
  • RSS feeds and article URLs are fetched server-side, bypassing CORS restrictions
  • The Cloudflare Worker validates a secret token (wc-9f2k4p8m) on every request

3. Using the generator

Tab: News Feed (default)

Loads live headlines from BBC Sport, Sky Sports, The Athletic and Guardian. Switch sports using the icon bar at the top.

  • Click any headline to expand it
  • Optionally upload a supporting file (PDF, DOCX, TXT) — treated as trusted fact by the AI
  • Optionally paste a URL in the Extra Context field (e.g. a Wikipedia page, Sky Sports article, club stats page) — click Add and the worker fetches it server-side. Can be combined with a file upload.
  • Click Generate WhatChan Article — the 6-step pipeline runs automatically

Tab: Paste URL

Enter any article URL, click Fetch — the worker extracts the article text. Choose a category, optionally add notes or an uploaded file, then generate.

Tab: Evergreen

Pre-built topic templates for each sport. Select a team and article type (Match Preview, Match Review, Fixtures, Season Preview, etc.) or choose from 15 evergreen topics. Supports batch generation.

Tab: Generated

All articles generated in this session — and any from the past 7 days that haven't been deleted. Articles now persist across page refreshes and browser restarts. Each card shows when it was saved. Use Copy HTML to paste into WordPress, DOCX to download a Word file, or Delete once you've used it.

File upload

Available on: News Feed articles, Paste URL, Evergreen/custom tabs. Accepts PDF, DOCX, TXT. Content is labelled as TRUSTED ADDITIONAL CONTEXT — the AI treats it as fact, not something to verify or question. Good uses: a player's stats sheet, a second source article, club press releases.

PDF NOTE Only text-based PDFs work. If you print-to-PDF or "Save as PDF" from a browser, the text is extractable and works fine. Scanned PDFs (photographed documents) contain images of text — the tool will show a red error and 0 chars extracted. In that case, use the URL context field instead, or save the content as a .txt or .docx file.

Article length

Four options: Standard (~700 words), Medium (~1,100 words), Longform (~1,800 words), Custom (enter your own word count).

5. The generation pipeline

Every article goes through these steps automatically. Watch the pipeline bar and Activity Log for progress.

1
Fetch — The worker retrieves the full article text from the source URL. Any uploaded files or URL context you added are merged in here as trusted facts.
2
Web Verify (Perplexity) — Perplexity Sonar Pro runs a live web search to verify current facts for the article topic: current manager, current club of each player, playing status, recent results, league position. This runs before writing starts. It's the primary defence against AI models using stale training data (e.g. "Estêvão is at Palmeiras" when he's been at Chelsea for months).
3
Analyse — Gemini 2.5 Flash reads the source and extracts key facts, names, quotes, stats, and the main angle. The web-verified facts are injected here as authoritative ground truth.
4
Write — Your selected model writes a completely original article. The writer is given both the analysed source facts and the web-verified current facts, and is explicitly instructed not to introduce any time-sensitive information not in those sources. Truncated quotes (from RSS feeds cut at 200 chars) are blocked — the AI must omit or paraphrase them.
4b
Hallucination scan — Gemini 2.5 Flash compares the draft against the source + verified facts. Checks for: invented names, fabricated quotes, truncated quotes, invented stats, wrong current managers, wrong player clubs, wrong playing status. Suspect content is flagged and removed.
5
QC + SEO — Source-traceability check (can every claim be traced to the source or verified facts?) runs first. If it fails, the article is automatically rewritten with the fixes applied, then a second QC pass runs. SEO fields are then generated: 5 title options, 2 meta descriptions, up to 10 tags, 3 focus keyphrases, URL slug.
6
E-E-A-T Polish — Your selected model reviews and strengthens Experience, Expertise, Authoritativeness, Trustworthiness signals. Enhances pullquotes, stat cards, and source attributions.

The final output includes: full article HTML, schema.org Article + Breadcrumb JSON-LD, SEO data block, banners (WhatChan only), and a tag strip.

Article badges

Each generated article in the Generated tab shows one of two quality badges:

  • ✓ Web-Verified — Perplexity verified facts, hallucination scan clean, QC passed. Safe to publish.
  • ⚠ REVIEW BEFORE PUBLISHING — Either the hallucination scan flagged something, or QC failed even after a rewrite. Read carefully before publishing.

White label articles also show a White Label badge.

7. Saved articles

Generated articles are automatically saved in your browser and survive page refreshes, tab closes, and browser restarts. You can generate a batch of 10, come back the next day, and they'll all be there.

  • Articles are saved in your browser's localStorage — they are specific to this browser and device
  • Articles expire automatically after 7 days — they are pruned on page load
  • Each card shows when it was saved: saved today, saved yesterday, saved 3d ago
  • Click Delete on a card to remove it immediately (after you've published it or decided not to use it)
NOTE localStorage is per-browser, per-domain. If you clear browser data or use a different browser/device, articles are gone. The WordPress HTML is in the copied output — always Copy HTML anything you want to keep before clearing browser data.

8. White Label mode

The White Label checkbox sits in the bar directly below the model and article length controls. Tick it to reveal the white label options.

White Label When active, all articles are generated without WhatChan branding — no banners, no WhatChan mentions, schema uses your domain. A blue White Label badge appears on the article card.

Domain

Enter the client's domain (e.g. sportsblog.co.uk). All schema.org URLs (article URL, publisher URL, breadcrumb) will use https://sportsblog.co.uk/blog/... instead of the WhatChan URL. Leave blank and the schema URLs will use example.com as a placeholder.

Author name

Enter the author's name — this replaces Adrian Dane in the article byline and schema.org author field.

Leave blank for fully anonymous: no author credit anywhere in the article, no Editor's Note section, no author in the schema. The AI is explicitly instructed not to include any byline.

Banners

Both banners (the sport TV banner mid-article and the blog banner at the bottom) are automatically removed for all white label articles — you do not need to do anything. The generated HTML will contain only the article content.

Workflow for white label articles

  1. Tick White Label
  2. Enter the client's domain and author name (or leave author blank for anonymous)
  3. Generate the article as normal
  4. Use Copy HTML for WordPress, or DOCX for a Word document
  5. When done, untick White Label to return to WhatChan mode

9. DOCX export

Every article card has a DOCX button. Click it to download the article as a .docx Word file, named after the URL slug.

The Word document contains a clean version of the article:

  • Headings, paragraphs, bold, italic — standard Word formatting
  • Pull quotes converted to Word blockquotes
  • Editor's Note converted to italic paragraph

The following are stripped from the DOCX (they're WordPress-specific):

  • Schema.org JSON-LD scripts
  • SEO comment block
  • CSS style blocks
  • Sport TV banner and blog banner
  • Stats strip cards
  • Tag strip
TIP DOCX export works for both WhatChan articles and white label articles. For white label, the DOCX is the cleanest option — no banners were ever added, so you get pure article content ready to hand off to a client.
NOTE The DOCX library loads from a CDN when the page first opens. If you click DOCX immediately after opening the page, it may say "not loaded yet — please wait a moment". Wait 2-3 seconds and try again.

NOTE The secret token is visible in the HTML source (it has to be, so the browser can send it). Your GitHub repo is public (required for free GitHub Pages). The token is your first line of defence, the spend cap is your backstop.

12. Making changes and deploying

Changing the frontend (docs/index.html)

Edit the file, then:

git add docs/index.html
git commit -m "Description of change"
git push

Wait ~2 minutes, then test in an incognito window (Ctrl+Shift+N) to avoid seeing a cached version.

Changing the worker (worker/src/index.js)

cd worker
npx wrangler deploy

This deploys immediately — no waiting.

Updating worker secrets

cd worker
npx wrangler secret put OPENROUTER_API_KEY
npx wrangler secret put WC_SECRET

Viewing worker logs (for debugging)

cd worker
npx wrangler tail

Adding a new AI model to the dropdown

Find the var MODELS = [ array in docs/index.html. Add a new entry:

{ id: 'provider/model-name', label: 'Display Name', free: false, costIn: 0.001, costOut: 0.004 }

Model IDs come from openrouter.ai/models — use the ID shown under each model name.

Checking what went wrong

The Activity Log (bottom left of the tool) shows every step. Look for lines marked in red — they contain the exact error message from OpenRouter. Key things to spot:

  • [API] model=... — confirms which model is actually being used
  • Web research unavailable: ... — Perplexity verify step failed (article continues without it, but check the REVIEW badge)
  • Hallucinations detected (N) — shows the scan found and flagged invented content
  • QC FAIL — rewriting with fixes — automatic rewrite triggered; a second QC pass will follow
  • Rate limited (attempt X/5) — retrying automatically, wait for it
  • [White Label] in the starting log line — confirms white label mode was active for that run
