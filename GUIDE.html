2. How it works (architecture)

Your browser never talks to OpenRouter directly. Everything goes through a Cloudflare Worker that holds your API key securely.

Your Browser (GitHub Pages)
    │
    ├── GET  /api/rss          → Cloudflare Worker → RSS feeds (BBC, Sky, etc.)
    ├── POST /api/fetch-article → Cloudflare Worker → Article URL (extracts text)
    └── POST /api/messages     → Cloudflare Worker → OpenRouter API → AI model

This means:

  • Your OpenRouter API key is never visible in the browser or page source
  • RSS feeds and article URLs are fetched server-side, bypassing CORS restrictions
  • The Cloudflare Worker validates a secret token (wc-9f2k4p8m) on every request

3. Using the generator

Tab: News Feed (default)

Loads live headlines from BBC Sport, Sky Sports, The Athletic and Guardian. Switch sports using the icon bar at the top.

  • Click any headline to expand it
  • Optionally upload a supporting file (PDF, DOCX, TXT) — treated as trusted fact by the AI
  • Click Generate WhatChan Article — the 5-step pipeline runs automatically

Tab: Paste URL

Enter any article URL, click Fetch — the worker extracts the article text. Choose a category, optionally add notes or an uploaded file, then generate.

Tab: Evergreen

Pre-built topic templates for each sport. Select a team and article type (Match Preview, Match Review, Fixtures, Season Preview, etc.) or choose from 15 evergreen topics. Supports batch generation.

Tab: Generated

All articles generated in this session. Click any to preview the full HTML output. Use Copy HTML to paste into WordPress (Gutenberg custom HTML block).

File upload

Available on: News Feed articles, Paste URL, Evergreen/custom tabs. Accepts PDF, DOCX, TXT. Content is labelled as TRUSTED ADDITIONAL CONTEXT — the AI treats it as fact, not something to verify or question. Good uses: Trippier's stats sheet, a second source article, club press releases.

Article length

Four options: Standard (~700 words), Medium (~1,100 words), Longform (~1,800 words), Custom (enter your own word count).

5. The generation pipeline

Every article goes through these steps automatically. Watch the Activity Log for progress.

1
Fetch — The worker retrieves the full article text from the source URL (or you've already provided source text/uploads).
2
Analyse — Gemini 2.5 Flash reads the source and extracts key facts, names, quotes, stats, and the main angle. This is what the writer uses as its ground truth.
3
Write — Your selected model writes a completely original WhatChan article using the extracted facts. Never a copy — a new angle, new structure, expert analysis added.
2b
Hallucination scan — Gemini 2.5 Flash compares the draft against the source. Any invented names, fabricated quotes, invented stats, or placeholder text ([insert date] etc.) are automatically removed.
4
QC + SEO — Fact-check pass (wrong names, wrong stats, outdated info) runs first, then SEO fields (5 title options, 2 meta descriptions, tags, keyphrases, URL slug) are generated.
5
E-E-A-T Polish — Your selected model reviews and strengthens Experience, Expertise, Authoritativeness, Trustworthiness signals. Enhances pullquotes, stat cards, and source attributions.

The final output includes: full article HTML, schema.org Article + Breadcrumb JSON-LD, SEO data block, two banners (sport banner mid-article, blog banner at the bottom), and a tag strip.

6. Managing OpenRouter credits

ACTION REQUIRED Keep credits topped up. If your balance hits zero, the generator will fail silently with API errors. Check your balance at openrouter.ai/credits.

Cost guide

  • With Gemini 2.5 Flash as the writing model, each article costs roughly £0.01-0.02 total (including all pipeline steps)
  • With Claude Opus 4, expect £0.10-0.15 per article
  • £10 in credits = roughly 500-1000 articles with the default model

Spend cap (do this now if not done)

  1. Go to openrouter.ai/settings/limits
  2. Set a monthly spending limit (e.g. £10)
  3. This protects you if anyone finds the tool URL and tries to abuse it

Adding credits

  1. Go to openrouter.ai/credits
  2. Click Add credits and pay by card
  3. Credits are available immediately — no need to redeploy anything

7. Security

RiskProtection in place
API key leaked to browserKey stored only as Cloudflare Worker secret — never in the HTML
Someone finds the URL and uses your creditsWorker secret token (wc-9f2k4p8m) — requests without it are rejected
Unlimited abuse if token is foundOpenRouter monthly spend cap (set this yourself)
Other websites embedding the toolCORS ALLOWED_ORIGINS blocks cross-site requests
NOTE The secret token is visible in the HTML source (it has to be, so the browser can send it). Your GitHub repo is public (required for free GitHub Pages). The token is your first line of defence, the spend cap is your backstop.

If you suspect abuse

  1. Check your OpenRouter usage at openrouter.ai/activity
  2. Change the token: update WC_TOKEN in docs/index.html, run npx wrangler secret put WC_SECRET with the new value, and push

8. Making changes and deploying

Changing the frontend (docs/index.html)

Edit the file, then:

git add docs/index.html
git commit -m "Description of change"
git push

Wait ~2 minutes, then test in an incognito window (Ctrl+Shift+N) to avoid seeing a cached version.

Changing the worker (worker/src/index.js)

cd worker
npx wrangler deploy

This deploys immediately — no waiting.

Updating worker secrets

cd worker
npx wrangler secret put OPENROUTER_API_KEY
npx wrangler secret put WC_SECRET

Viewing worker logs (for debugging)

cd worker
npx wrangler tail

Adding a new AI model to the dropdown

Find the var MODELS = [ array in docs/index.html (~line 630). Add a new entry:

{ id: 'provider/model-name', label: 'Display Name', free: false, costIn: 0.001, costOut: 0.004 }

Model IDs come from openrouter.ai/models — use the ID shown under each model name.

9. Troubleshooting

SymptomLikely causeFix
Pipeline failed: Rate limited Free model selected, or too many requests too fast Select a paid model from the dropdown. Wait 60 seconds and retry.
Pipeline failed: No endpoints found for... Model ID is wrong or that model was removed from OpenRouter Check the model ID at openrouter.ai/models. Update the MODELS array.
Pipeline failed: Unauthorised WC_SECRET worker secret not set or wrong value Run npx wrangler secret put WC_SECRET and enter wc-9f2k4p8m
Pipeline failed: Missing Authentication OPENROUTER_API_KEY secret not set on worker Run npx wrangler secret put OPENROUTER_API_KEY and paste your key
Old version of site showing after push GitHub Pages cache Open in incognito (Ctrl+Shift+N) or press Ctrl+Shift+R to hard-refresh
News feed not loading RSS feeds temporarily down, or worker error Click the refresh button. If all feeds fail, check the worker is deployed.
Article has [insert date] or invented names Hallucination scan missed it, or model misbehaved Select a higher-quality model. The hallucination scan catches most cases but is not 100%.
OpenRouter credits ran out mid-article Balance hit zero Top up at openrouter.ai/credits. Set a spend alert so this doesn't happen again.

Checking what went wrong

The Activity Log (bottom left of the tool) shows every step. Look for lines marked in red — they contain the exact error message from OpenRouter. Key things to spot:

  • [API] model=... — confirms which model is actually being used
  • Hallucinations detected (N) — shows the scan found and removed invented content
  • Rate limited (attempt X/5) — retrying automatically, wait for it
